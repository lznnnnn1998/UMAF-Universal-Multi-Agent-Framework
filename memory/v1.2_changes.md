## v1.2 Changes (May 2026)

**Why:** The research pipeline produced empty results because workers used nested `claude -p` calls (the agent IS already `claude -p`, so nesting created recursive invocations that timed out). Additionally, global `Bash(*)`/`Write(*)`/`Edit(*)` permissions were too broad, and `claude_env_sample.json` with real API keys was tracked in git.

**How to apply:** When modifying the framework, ensure backend-aware task generation is maintained — `claude_cli` agents should never be instructed to use `call_claude`. Keep permissions scoped to the working directory in `.claude/settings.local.json`. Use the conversation logger (`agent_name` param) for debugging.

### Changes

**Agent Loop (`agent.py`):**
- `_save_agent_log()` function saves prompt + response + success + elapsed to `agent_logs/<name>_<timestamp>.json`
- `run_agent()` accepts `agent_name` parameter
- `call_claude` spec now says "Do NOT spawn nested claude -p calls" instead of encouraging them
- System prompt no longer says "Use Bash for any command-line operation"

**LLM (`llm.py`):**
- `ClaudeCLILLM.timeout` increased from 120s to 300s

**Research Pipeline:**
- `research/head_agent.py`: Backend-aware — `claude_cli` uses Read-only tools (no WebSearch, no Bash). Pure reasoning decomposition.
- `research/worker_agent.py`: Backend-aware task generation via `_build_worker_task_claude_cli()` and `_build_worker_task_deepseek()`. `claude_cli` workers use WebSearch + Write + Read directly.
- `research/graph.py`: Max concurrent workers reduced from 4 to 2.
- All agents pass `agent_name` to `run_agent()`.

**Permissions (`.claude/settings.local.json`):**
- Scoped Bash: `curl *`, `ls *`, `cat *`, `mkdir *`, `wc *` (safe, read-only or local-dir-only commands)
- Scoped Read/Write/Edit for `research_output/**`
- Global `.claude/settings.json` only has `Read(*)`, `WebSearch(*)`, `WebFetch(*)`
- These permissions apply to both the parent Claude Code AND `claude -p` subprocesses (subprocess walks up from cwd to find project `.claude/`)

**Security:**
- `claude_env_sample.json` removed from git tracking (was tracked in initial commit). Replaced with `claude_env_sample.example.json` containing placeholder `YOUR_DEEPSEEK_API_KEY`.
- `claude_config.py` auto-falls back to `.example.json` if main config doesn't exist.
- `.gitignore`: added `research_output/`

**Verified Results:**
- 4/6 workers produce 21-26KB research files
- Top score: 43/50 (Flash Attention, Ring Attention)
- LaTeX: 41KB with equations, tables, citations
- 2/6 workers still hit 300s timeout (produce files but `TASK_COMPLETE` not detected)
